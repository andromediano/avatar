# Feature: 신체 치수 입력

사용자가 자신의 신체 치수를 입력하여 3D 아바타의 체형을 맞춘다.
직접 입력, 사이즈 라벨 선택, 사진 기반 추정(MediaPipe Pose) 세 가지 방식을 지원한다.

---

## Background

```gherkin
Given 사용자가 가상 피팅 페이지에 있다
And 아바타 모델이 로드되어 있다
```

---

## 직접 입력

### Scenario: 슬라이더로 체형 파라미터 조절

```gherkin
Given 측정값 입력 패널이 표시되어 있다
When 사용자가 "가슴둘레" 슬라이더를 0.7 위치로 드래그한다
Then 아바타의 chest morph target이 0.7로 실시간 변경된다
And 슬라이더 옆에 현재 값(0.7)이 표시된다
```

### Scenario: 모든 필수 파라미터 슬라이더 제공

```gherkin
Given 측정값 입력 패널이 표시되어 있다
Then 다음 슬라이더들이 모두 존재한다:
  | 라벨       | 파라미터 | 범위    | 기본값 |
  | 키         | height  | 0 ~ 1  | 0.5   |
  | 가슴둘레   | chest   | 0 ~ 1  | 0.5   |
  | 허리둘레   | waist   | 0 ~ 1  | 0.5   |
  | 엉덩이둘레 | hip     | 0 ~ 1  | 0.5   |
  | 체중       | weight  | 0 ~ 1  | 0.5   |
And 각 슬라이더의 step은 0.01이다
```

### Scenario: cm 단위 직접 수치 입력

```gherkin
Given 측정값 입력 모드가 "직접 입력"이다
When 사용자가 다음 값을 입력한다:
  | 항목     | 값 (cm/kg) |
  | 키       | 175        |
  | 가슴둘레 | 95         |
  | 허리둘레 | 80         |
  | 엉덩이둘레 | 97       |
  | 체중     | 72         |
Then 입력값이 정규화되어 0~1 범위의 morph target 값으로 변환된다
And 아바타 체형이 해당 치수에 맞게 변형된다
```

### Scenario: 입력값 범위 검증

```gherkin
Given 측정값 입력 모드가 "직접 입력"이다
When 사용자가 키에 250을 입력한다
Then "키는 140~200cm 범위여야 합니다" 경고가 표시된다
And 입력값이 아바타에 반영되지 않는다
```

## 사이즈 라벨 선택

### Scenario: 남성 사이즈 라벨로 체형 설정

```gherkin
Given 측정값 입력 모드가 "사이즈 선택"이다
And 성별이 "남성"으로 선택되어 있다
When 사용자가 사이즈 "100 (L)"을 선택한다
Then KS K 0051 기준 다음 치수가 자동 설정된다:
  | 항목       | 값 (cm) |
  | 키         | 175     |
  | 가슴둘레   | 99.5    |
  | 허리둘레   | 85.5    |
  | 엉덩이둘레 | 98      |
  | 어깨너비   | 45      |
And 아바타 체형이 해당 치수에 맞게 변형된다
```

### Scenario: 여성 사이즈 라벨로 체형 설정

```gherkin
Given 측정값 입력 모드가 "사이즈 선택"이다
And 성별이 "여성"으로 선택되어 있다
When 사용자가 사이즈 "90 (M)"을 선택한다
Then KS K 0051 기준 여성 90 호칭의 표준 치수가 자동 설정된다
And 아바타 체형이 해당 치수에 맞게 변형된다
```

### Scenario: 사이즈 라벨 선택지 표시

```gherkin
Given 성별이 "남성"으로 선택되어 있다
Then 다음 사이즈 옵션이 표시된다:
  | 호칭 | 레터 | 가슴둘레 범위 |
  | 85   | XS   | 82–87 cm    |
  | 90   | S    | 87–92 cm    |
  | 95   | M    | 92–97 cm    |
  | 100  | L    | 97–102 cm   |
  | 105  | XL   | 102–107 cm  |
  | 110  | XXL  | 107–112 cm  |
```

## 사진 기반 추정 (MediaPipe Pose)

### Scenario: 정면 사진으로 키포인트 추출

```gherkin
Given 측정값 입력 모드가 "사진 추정"이다
And 사용자가 키(cm)를 참조값으로 입력했다
When 사용자가 정면 전신 사진을 업로드한다
Then MediaPipe Pose heavy 모델이 로드된다
And 사진에서 33개 키포인트가 추출된다
And 키포인트가 사진 위에 오버레이로 표시된다
And "측면 사진도 업로드해주세요" 안내가 표시된다
```

### Scenario: 정면 + 측면 사진으로 신체 치수 추정

```gherkin
Given 정면 사진에서 키포인트가 추출되었다
When 사용자가 측면 전신 사진을 업로드한다
Then 측면 사진에서도 33개 키포인트가 추출된다
And 정면/측면 키포인트 비율과 참조 키(cm)를 기반으로 다음 치수가 추정된다:
  | 항목       | 추정 방법                          |
  | 어깨너비   | 정면 좌우 어깨 키포인트 간 거리     |
  | 팔길이     | 어깨-팔꿈치-손목 키포인트 연결 길이 |
  | 다리길이   | 골반-무릎-발목 키포인트 연결 길이   |
  | 가슴둘레   | 정면 폭 + 측면 깊이 기반 타원 둘레 추정 |
  | 허리둘레   | 정면 폭 + 측면 깊이 기반 타원 둘레 추정 |
  | 엉덩이둘레 | 정면 폭 + 측면 깊이 기반 타원 둘레 추정 |
And 추정된 치수가 입력 폼에 자동 채워진다
And 아바타 체형이 추정 치수에 맞게 변형된다
```

### Scenario: 사진에서 사람이 감지되지 않음

```gherkin
Given 측정값 입력 모드가 "사진 추정"이다
When 사용자가 사람이 없는 사진을 업로드한다
Then MediaPipe Pose가 키포인트 0개를 반환한다
And "사진에서 사람을 감지할 수 없습니다. 전신이 보이는 사진을 업로드해주세요." 오류가 표시된다
```

### Scenario: 전신이 보이지 않는 사진

```gherkin
Given 측정값 입력 모드가 "사진 추정"이다
When 사용자가 상반신만 보이는 사진을 업로드한다
Then 키포인트 중 하반신(무릎, 발목) visibility가 낮다
And "전신이 보이는 사진을 업로드해주세요. 하반신이 잘 보이지 않습니다." 경고가 표시된다
```

### Scenario: 추정 결과 수동 보정

```gherkin
Given 사진 기반 추정이 완료되었다
And 추정 치수가 입력 폼에 채워져 있다
When 사용자가 "허리둘레" 값을 수동으로 수정한다
Then 수정된 값이 아바타 체형에 즉시 반영된다
And 다른 추정값은 유지된다
```

### Scenario: 온디바이스 처리 확인

```gherkin
Given 측정값 입력 모드가 "사진 추정"이다
When 사진 추정 과정이 실행된다
Then 모든 처리가 브라우저 내에서 완료된다 (WASM)
And 사진 데이터가 서버로 전송되지 않는다
And 처리 완료 후 원본 사진 데이터가 메모리에서 해제된다
```

## 입력 모드 전환

### Scenario: 입력 모드 탭 전환

```gherkin
Given 측정값 입력 패널이 표시되어 있다
Then 3개의 입력 모드 탭이 존재한다: "슬라이더" | "사이즈 선택" | "사진 추정"
When 사용자가 "사이즈 선택" 탭을 클릭한다
Then 사이즈 선택 UI가 표시된다
And 이전 입력값은 유지된다
```

## 데이터 저장

### Scenario: 입력된 치수를 로컬에 저장

```gherkin
Given 사용자가 체형 파라미터를 설정했다
When 사용자가 "저장" 버튼을 클릭한다
Then 치수 데이터가 IndexedDB에 암호화(AES-GCM)되어 저장된다
And "치수가 저장되었습니다" 확인 메시지가 표시된다
```

### Scenario: 저장된 치수 불러오기

```gherkin
Given 사용자가 이전에 치수를 저장한 적이 있다
When 가상 피팅 페이지에 재접속한다
Then IndexedDB에서 저장된 치수가 복호화되어 로드된다
And 아바타가 저장된 체형으로 자동 설정된다
```

## 기술 사양

| 항목 | 값 |
|------|-----|
| MediaPipe 모델 | pose_landmarker_heavy (사진 추정용) |
| MediaPipe 패키지 | @mediapipe/tasks-vision |
| 추론 방식 | 온디바이스 (WASM, GPU delegate) |
| 키포인트 수 | 33개 |
| 라이선스 | Apache 2.0 |
| 치수 저장 | IndexedDB + Web Crypto API (AES-GCM) |
