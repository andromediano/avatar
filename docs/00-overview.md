# Non-AI 3D 아바타 가상 피팅 — 프로젝트 개요

## 1. 배경

기존 AI VTON(Virtual Try-On)은 Diffusion 모델로 2D 이미지를 생성하는 방식이다.
이 문서군은 **AI를 사용하지 않는 대안적 접근법** — 3D 아바타에 의류를 물리 시뮬레이션으로 입히는 방식 — 의 기술 스펙을 조사한다.

---

## 2. AI VTON vs 3D 아바타 방식 비교

| 항목 | AI VTON (2D Diffusion) | 3D 아바타 + 물리 시뮬레이션 |
|------|----------------------|--------------------------|
| **실시간성** | 5-60초/이미지 | 30-60 FPS (로딩 후) |
| **상호작용** | 없음 (결과 이미지 1장) | 360° 회전, 포즈 변경, 색상 변경 |
| **정확도** | 사실적 외관, 물리 부정확 | 물리적으로 정확한 핏/실루엣 |
| **인프라 비용** | GPU 서버 필수 (A100 등) | 클라이언트 사이드 (브라우저) |
| **의류 에셋 비용** | 사진 1장이면 충분 | 3D 모델링 필수 (의류당 수시간) |
| **개발 난이도** | 모델 학습/파인튜닝 | 3D 파이프라인 + 물리 엔진 |
| **오프라인** | 불가 (서버 필요) | 가능 (PWA) |
| **확장성** | 이미지당 비용 발생 | 초기 에셋 비용 후 추가 비용 없음 |

### 핵심 트레이드오프

- **AI VTON**: 의류 에셋 비용 최소 → 빠른 MVP, 하지만 실시간 상호작용 불가
- **3D 아바타**: 실시간 상호작용 + 제로 서버 비용 → 장기적 확장성, 하지만 의류 3D 모델링 필요

## 3. 3D 아바타 방식의 핵심 기술 스택

```
사용자 신체 치수 입력 (직접 입력 / 사이즈 선택 / MediaPipe Pose 사진 추정)
    ↓
[Body Measurement Mapping] ← 치수 → 아바타 파라미터 변환 (MediaPipe Pose로 사진 기반 추정 지원)
    ↓
[3D Body Model] ← makehuman-js, SMPL, glTF Avatar
    ↓
[Garment Loading] ← glTF/GLB 의류 모델
    ↓
[Cloth Simulation] ← Ammo.js, WebGPU Compute, Pre-baked
    ↓
[Rendering] ← Three.js + WebGPU (WebGL 폴백)
    ↓
브라우저에서 실시간 3D 피팅 결과
```

## 6. 기술 선택 요약

| 기술 영역 | 추천 선택지 | 대안 |
|-----------|------------|------|
| 인체 모델 | makehuman-js / glTF 아바타 | SMPL (Python 백엔드) |
| 천 시뮬레이션 | Ammo.js (MVP) / WebGPU (고품질) | Pre-baked (Blender) |
| 렌더링 | Three.js + WebGPU (WebGL 폴백) | Babylon.js |
| 바디 트래킹 | MediaPipe Pose (채택) | WebXR Body Tracking |
| 사진 신체 추정 | MediaPipe Pose heavy (채택) | SMPL 서버 사이드 |
| 의류 포맷 | glTF/GLB | OBJ, FBX |
| 의류 제작 | Blender (무료) | CLO3D (유료) |
| 프론트엔드 | React + React Three Fiber | Vue + TresJS |

## 7. AI 방식과의 하이브리드 가능성

두 방식은 상호 배타적이지 않다:

- **3D 아바타**: 실시간 탐색, 360° 뷰, 사이즈 추천
- **AI VTON**: 최종 "내 사진에 입어보기" 이미지 생성

> 사용자가 3D 아바타로 의류를 탐색 → 마음에 드는 것을 AI VTON으로 자기 사진에 합성하는 2단계 UX가 가능하다.
